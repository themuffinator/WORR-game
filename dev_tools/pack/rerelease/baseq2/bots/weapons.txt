// Overview Of Weapon Properties:

// name: 				classname of the weapon, as it appears in C code.
// damage:				the amount of raw damage this weapon does each "shot"
// ideal_fov:			the ideal FOV the target should be inside of for the bot to fire the weapon
// min_range: 			the absolute min range the bot must be from it's enemy to use this weapon ( else it is invalid ). 0 if don't care
// max_range: 			the absolute max range the bot can be from it's enemy to use this weapon ( else it is invalid ). 0 if don't care
// min_height			the absolute min height difference ( below the bot ) between the bot and the enemy allowed. 0 if don't care
// max_height			the absolute max height difference ( above the bot ) between the bot and the enemy allowed. 0 if don't care
// speed:				if fire a projectile, how fast the projectile travels ( for predictive aiming )
// priority: 			relative priority that you assign this weapon ( vs. other valid weapons ) - tweak this as you like. 0 = never choose this weapon
// ammo_name: 			classname of the ammo, as it appears in C code. Use "none" if no ammo needed
// min_ammo: 			the minimum amount of ammo this weapon needs to have to be considered for use
// max_ammo:			the maximum amount of ammo the player can carry for this weapon
// flags: 				describe properties of this weapon to the bots
// aim_point: 			where on the enemies body we should aim the weapon.
// trigger_type:		how the bots should handle the trigger to fire this weapon
// trigger_hold:		how long to hold trigger, in seconds, before weapon is ready to fire. 0 = instant
// trigger_cooldown:	how long to wait, in seconds, before can fire again. 0 = no cooldown


// Valid Aim Points:

// feet:				aim at the targets feet
// head:				aim at targets head
// center:				aim at targets center ( default )


// Valid Weapon Flags:

// melee:				this is a melee weapon
// hitscan:				this is an instant hit weapon
// projectile:			this weapon fires projectiles
// parabolic:			this weapon is affected by gravity
// explosive:			this weapon is explosive
// sticky:				this weapon shot sticks to walls
// starting:			bot spawns with this weapon in inventory
// initial:				bot spawns with this weapon in hand
// suitable:			this weapon is suitable for general combat, else bot will focus on finding a suitable gun 
// rocket_jump:			this weapon could be used for rocket_jump traversals


// Valid Trigger Types:

// continuous:			fire this weapon by always holding down the trigger
// hold_and_release:	fire this weapon by holding the trigger for a time, then releasing it


// For A Weapon To Be Considered, Each Of The Following Must Be True:
 
// 1. The bot has the weapon in inventory, with enough ammo.
// 2. The bots target is within the min/max distance from the bot.
// 3. If min/max height is defined, the enemy must be within this distance on the Z axis ( up/down in idTech ).
// 4. If all of the above are true, then the weapon with the highest priority value wins.
